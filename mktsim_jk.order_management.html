add_order(side, symbol, quantity, limit)[source]

The add_order function takes in a side, symbol, quantity and limit. It then creates an order object with the given parameters. If the symbol is not already in self.orders it adds it to self.orders as a key with an empty list as its value (value TBD). Finally, it appends the order to that symbols list of orders and returns that order.

Parameters:

  • self – Represent the instance of the class

  • side – Determine whether the order is a buy or sell

  • symbol – Identify the stock that is being traded

  • quantity – Determine how many shares of a stock the user wants to buy or sell

  • limit – Set the limit price of an order

Returns:

The order that was just added

check_tick(symbol, bid, ask)[source]

The check_tick function is called every time a new tick comes in and is used as the callback function.

It checks the bid and ask prices against the limit price of each order, and if they are within range, it executes that order. The execution_client then sends an instruction to buy or sell at that price.

# Given a tick, will compare to available orders in list

Parameters:

  • self – Refer to the object itself

  • symbol – Check the orders dictionary for a symbol

  • bid – Compare the bid price to the order limit

  • ask – Compare the ask price to the limit price of an order

Returns:

Nothing
